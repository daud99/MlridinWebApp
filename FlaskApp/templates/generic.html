Introduction

Intrusion Detection Systems are used to detect any malicious or anomalous data in the network traffic. Basic idea is to classify network traffic into either normal or ma-licious. This thesis aims to address the issues of anomaly-based network intrusion and an approach to classify network data using machine learning-based techniques. A lot of research has been done on using machine learning and artificial intelligence for network intrusion detection systems. This project focuses on network traffic fea-tures extracted from CICFlowMeter that have been identified as the most important features in recent research.

CICFlowMeter is an open-source tool intended to extract 84 network traffic features proposed in the paper. These features as published in this paper representing some good features of the network traffic. This tool takes the traffic capture file and generates a CSV file based on those 84 features. These features are based on the OSI layer 3 and layer 4 heading information and other time and volume-based infor-mation of the network traffic such as the number of bytes, packets, durations, etc for both forward and backward directions. Layer 3 header specifies source IP, destina-tion IP, and type of service whereas layer 4 has lots of header flags including source and destination ports, sequence number, acknowledgment number, and other flags like URG, ACK, PSH, RST, SYN, and FIN. It generates both TCP and UDP flows. UDP flow mostly ends with the flow timeout whereas TCP flows either end with FIN packet or flow timeout. Default flow timeout is 120000000 microseconds that is 120 seconds.
